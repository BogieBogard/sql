About This Tool

Running a read operation before your write helps prevent mistakes. This modifies UPDATE and DELETE queries into SELECT statements using regex. Allowing you to preview the data that will be targeted.

Future Vision

Ideally, a utility like this would be ran by default in a SQL editor. Where if an UPDATE or DELETE statement were executed by the user, then by default the editor would run a read version of the query and count the number of records that the pototenial writer operation would be changed. And if that record count was above a configured amount then the user would get a notice before the update/delete query ran prompting them, "This query will change X number of rows. Do you wish to continue?". This would be a great way to prevent accidental data changes, and would protect engineers from themselves. In the future, I'd like to add this feature to an open source SQL editor.
